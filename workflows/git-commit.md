このプロジェクトでの Git コミットに関する標準的な手順とルールです。

## ルール
- **コミットメッセージは原則として日本語で記述する。**
- 接頭辞（Angular 形式の prefix）を使用する場合は、それに続く説明文を日本語にする。
  - 例: `feat: 新しいバリデーション機能の追加`
  - 例: `fix: 請求書発行時の計算バグを修正`
  - 例: `chore: 依存パッケージの更新`

## 手順
1. 変更されたファイルを確認する。
2. 適切なファイルをステージングする (`git add`)。
3. 上記ルールに従って日本語でコミットメッセージを作成し実行する (`git commit`)。
