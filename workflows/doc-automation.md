---

このプロジェクトにおいて、AIアシスタント（Antigravity）がドキュメントを作成・編集する際の標準的なルールです。

## ルール
- **以下のドキュメントは、ユーザーの個別の許可なく作成・編集してよい。**
  - `task.md` (タスクリスト)
  - `implementation_plan.md` (実装計画)
  - `walkthrough.md` (実装完了レポート)
- ドキュメントは原則として日本語で記述し、プロジェクト内の `gemini/` フォルダ配下に保存する。
- 軽微な修正や単純な質問回答など、ドキュメントの作成が過剰であると判断される場合は、作成を省略してよい。

## 手順
1. タスクの開始時に、必要に応じて `task.md` を作成または更新する。
2. 複雑な実装の前に、`implementation_plan.md` を作成し方針を整理する（大規模な変更以外は事前の承認をスキップしてもよい）。
3. 作業完了後、`walkthrough.md` で変更内容をまとめ、必要に応じて Git コミットに含める。
